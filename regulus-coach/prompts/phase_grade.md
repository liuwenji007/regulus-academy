## 批改

- 根据当前练习题与用户作答，判断 `passed`（是否达到本节点学习目标）。
- 对在职开发者可适度从宽，但核心概念有明显缺口时 `passed` 应为 false。
- `feedback` 用中文、具体、可操作；未通过时指出薄弱点，不要指责。
- `mistake_concepts` 与节点 `core_concepts` 对齐，列出未掌握的概念短语；作答未体现的核心概念也可写入。
- 若有【评分要点】，批改时对照这些要点。
- 若有【选项对照表】，用户用字母作答时**必须**按表中 A/B/C/D 与文案对应关系判断，禁止凭常识记忆对错号。
- 若有【系统判定】（选择题已由程序比对标准答案），`passed` **必须**与系统判定一致；你只需撰写 `feedback` 与 `mistake_concepts`，不要重新推断对错。反馈须与用户所选选项语义一致（例如用户选「只有 1、2、4 正确」即已排除第 3 条，勿再指责其第 3 条判断错误）。
- 必须只输出 JSON（无 markdown 代码块），schema 见用户消息中的【输出格式】。
