# Regulus Academy — 设计理念

> 这些原则是在无数次自我否定和竞品体验中沉淀下来的。
> 不是凭空想出来的，而是被 DeepTutor 的配置劝退、被 OpenMAIC 的慢节奏拖到放弃之后，一点点挖出来的答案。

---

## 一、我们不是老师，是教练

老师站在讲台上，对着 50 个人讲同一堂课。教练坐在你旁边，看你的动作，告诉你"这里肩膀再低一点"。

**教练的两个核心行为：**
- **知道你现在在哪**——不给你推已经会的，也不给你推太难的；用户画像和学习目标会裁剪公共知识树，聚焦该练的部分
- **只纠正你最该纠正的那一个动作**——不是把教科书写进对话框

## 二、15 分钟是一个神圣的单位

不是 45 分钟，不是 2 小时，是 **15 分钟**。

因为通勤地铁是 15 分钟，午休间隙是 15 分钟，哄睡孩子之后的精力残值是 15 分钟。

一个 15 分钟包含：一个知识点 + 一个练习 + 一次反馈。做完了，大脑收到了明确的"完成"信号，可以安心去干别的。不会像 2 小时的视频课，永远卡在 43%。

> 15 分钟是用户场景的叙事单位，不写入 Prompt 计时——教练按节点边界推进，而不是掐表。

## 三、知识边界比知识库重要

我们不做知识库。不是因为技术不行，是因为不需要。

大模型已经学过 Go 并发、Agent 原理、RAG 架构。它缺的不是资料，是**边界**。

每个节点只定义三样东西：
- **该讲什么**（核心概念）  
- **容易错什么**（常见误区）  
- **不讲什么**（边界，避免跑题）

LLM 在这个框里自由发挥——讲解、出题、批改。内容是活的，边界是死的。

## 四、入口极简，进来就是核心体验

用户进来只需做一件事：输入想学的领域。Agent 问 1-2 句定边界，展示知识树，用户点「开始学习」。

不先让用户填问卷。不展示功能介绍页。不做新手引导教程。

**主路径只有三步：输入领域 → 选节点 → 对话学习。**

图谱、课程列表、设置（含 IM 频道）是辅助视图——用来看全局、管理多门课、接机器人，不挡这条主路。侧边栏只保留高频入口；教练对话和当前节点才是屏幕的主角。

Web 与 IM 共用同一份进度和会话记忆：终端上学到一半，换浏览器或手机继续，教练还是同一个。

**多学习角色：** 左下角可切换角色（如「工作 Go」「面试 Agent」）。每个角色有独立课程、进度与聊天；切换角色会刷新侧栏课程快捷，避免串号。

**完成与续学：** 节点通过后进入 `completed`。Web 用「继续 · 下一节」开新会话；IM 可说「下一节」。进行中的节点在 IM 发消息会续教练对话，不会被导航命令打断。

## 五、学完要能立刻用

我们不追求"系统性"，追求"可用性"。

今天学了 goroutine，晚上回家就能在项目里开一个协程。明天学了 channel，就能用 channel 重构上周写的共享变量。

用户不需要等到"学完整个 Go 并发"才有产出。每一个节点学完，都有一件事他今天就能干。

## 六、用户不知道自己在被"强化"

每个人学习都会犯错。传统的做法是告诉你"这道题你上次错了，再做一次"——用户立刻觉得自己很笨，想关掉。

我们的做法：Agent 悄悄把上次的错题埋进今天的练习里，换一套表述，换一种题型。用户以为在做新练习，实际上在被精准强化。

**无感强化**——让用户感觉自己在进步，而不是在被纠正。

## 七、知识树：主题模块 × 掌握深度

学习需要地图。没有地图，每节课都是孤岛，用户不知道自己走了多远、还剩多远。

知识树有两个**正交**的维度：

**主题模块（module）**——按「这一簇在讲什么」分组，例如「Goroutine 基础」「Channel 与通信」。推荐路径在同一模块内串联；换模块时，心智模型也跟着换。

**掌握深度（layer）**——按「你要学到多深」分三层：

- **🌱 入门**（~2 小时）：建立最基本的概念，搭建知识骨架。学完能看懂代码、能简单使用。
- **🌿 熟悉**（~8 小时）：挑实际生产中使用最多的技能，反复训练。学完能独立干活。
- **🌳 精通**（~20 小时）：深入底层原理与不常用的知识点。留给有追求的人。

同一个节点可以同时属于一个 module 和一个 layer。module 管「主题归属」，layer 管「难度层级」——两者独立，不互相替代。

**两种视图，各干一件事：**
- **知识图谱**（`#/graph`）——沉浸式看多领域、多模块的全局关系，适合「我在哪、还剩多少」
- **课程详情**（`#/tree/:id`）——列表按节点学，适合「点开就练」

用户一眼看到全貌，自己决定学到哪一层、先攻哪个主题。大部分人到「熟悉」就够了。

## 八、设计像工具，不像教室

一个屏幕只做一件事、进度可视化、即时反馈、游戏化但不幼稚。

我们不是一个"场景"，是一个"工具"。用户打开、学习、关闭，没有心理负担。

| 我们要做的 | 我们不要做的 |
|-----------|-------------|
| 主路径三步走，辅助页不抢戏 | 多层级菜单、功能介绍瀑布流 |
| 图谱看全局，列表按节点学 | 所有信息堆在一个页面 |
| 冷色调、高信息密度 | 温馨柔和、装饰性元素 |
| 打开即学，关闭即走 | 沉浸式课堂氛围 |
| 一个 Key 就能用 | 三步配置 API Key |
| 15 分钟一个闭环 | 45 分钟一节课 |
| Web / IM 进度一致 | 每个端各学各的 |

---

> 这些理念不是规则，是假设。每一个都需要被真实用户验证。
> 如果其中一条被证伪，那就改掉它——教练的第一个学生，永远是我们自己。
